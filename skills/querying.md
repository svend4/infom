# Навык: Запросы к графу знаний

## Режимы запросов

### local — конкретный вопрос
Ищет ближайшие узлы к вопросу. Лучше для:
- "Что такое X?"
- "Как работает Y?"
- "Перечисли компоненты Z"

```python
infom_query(question="Что такое нейронная сеть?", mode="local")
```

### global — обзор всего графа
Смотрит на все сообщества. Лучше для:
- "Какие главные темы в документах?"
- "Что встречается чаще всего?"
- "Дай общий обзор"

```python
infom_query(question="Какие ключевые концепции в документации?", mode="global")
```

### hybrid — комбинированный (рекомендуется)
Сначала local, потом обогащает глобальным контекстом. Лучше для:
- Сложных аналитических вопросов
- "Как X связано с Y?"
- "Есть ли противоречия?"

```python
infom_query(question="Как микросервисы связаны с безопасностью?", mode="hybrid")
```

## Интерпретация ответов

Без реального LLM (MockAdapter) ответы содержат геометрические описания:
- "Q6=37, форма triangle" — это адрес кластера в Q6-пространстве
- "скелет: tower" — тип фрактальной границы сообщества
- "ASCO архетип" — доминирующий тип узлов

С реальным LLM (Ollama/OpenAI) ответы становятся содержательными.
