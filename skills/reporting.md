# Навык: Форматирование результатов

## Шаблон отчёта

```markdown
# Анализ графа знаний InfoM
Дата: {YYYY-MM-DD}
Источник: {путь к документам}

## Статистика графа
- Нод: {N} | Рёбер: {E} | Сообществ: {C}
- Модульность Q: {Q} ({оценка: низкая <0.3 / средняя 0.3-0.5 / высокая >0.5})
- LP итераций: {iter}

## Сообщества знаний
| Форма | Q6 | Размер | Ключевые концепции |
|-------|-----|--------|-------------------|
| ...   | ... | ...    | ...               |

## Ответы на вопросы

### [Вопрос 1 — global]
{ответ infom_query}

### [Вопрос 2 — hybrid]
{ответ infom_query}

### [Вопрос 3 — local]
{ответ infom_query}

## Карта знаний (ASCII)
\`\`\`
{вывод infom_visualize}
\`\`\`

## Рекомендации
- [ ] {следующий шаг 1}
- [ ] {следующий шаг 2}

## Технические детали
- Адаптер LLM: {Mock / Semantic / Ollama / OpenAI}
- Токенов использовано: {сумма из ответов}
- Снапшот графа: graph_snapshots/latest.json
```

## Правила оформления
1. Всегда указывай дату и источник данных
2. Оценивай качество модульности словесно, не только числом
3. Уточняющие вопросы из infom_query сохраняй — они полезны для следующей итерации
4. Если Q < 0.2 — предупреди, что граф слабо структурирован
