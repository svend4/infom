# Навык: Индексация документов

## Правила

### Размер чанков
- Оптимально: 500–2000 символов на вызов infom_index()
- Длинные файлы разбивай на части по абзацам
- Не передавай бинарные файлы — только текст

### Форматы файлов
| Формат | Как обрабатывать |
|--------|-----------------|
| .md    | Читать как есть, markdown разметка не мешает |
| .txt   | Читать как есть |
| .py    | Извлекать docstrings + комментарии + названия функций/классов |
| .json  | Конвертировать в читаемый текст перед индексацией |
| .pdf   | Требуется внешний парсер (pdfplumber, pymupdf) |

### Порядок индексации
1. Сначала индексируй обзорные документы (README, intro, overview)
2. Затем детальные разделы
3. В конце — специфические детали и примеры

### Инкрементальная загрузка
```python
# ПРАВИЛЬНО — накапливать граф
infom_index(text=doc1, reset=False)
infom_index(text=doc2, reset=False)

# НЕПРАВИЛЬНО — каждый раз сбрасывать
infom_index(text=doc1, reset=True)  # теряет предыдущие данные
infom_index(text=doc2, reset=True)  # то же самое
```

### Проверка качества
После индексации запусти infom_stats() и проверь:
- Модульность Q > 0.3 — хорошая кластеризация
- Модульность Q < 0.1 — тексты слишком разрозненные или мало данных
- Нод > 50, рёбер > 10 — граф достаточно насыщен для анализа
